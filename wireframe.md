# 헬스맵 AI 모바일 앱 와이어프레임

## 1. 서비스 개요

**헬스맵 AI**는 현재 위치와 운동 목적을 바탕으로 가까운 헬스장과 공공체육시설을 추천하는 모바일 앱이다.

사용자는 출발 위치와 거리 범위를 설정한 뒤, 1차로 주변 운동시설을 탐색하고, 이후 선호 업종과 운동 목적을 선택하여 최종 추천 결과를 확인할 수 있다.

| 조건 고정 | 거리, 운동 목적, 선호 업종 저장 |
| 바로 검색 | 저장된 조건으로 1차 탐색 결과 이동 |

## 13. UI 톤앤매너

헬스맵 AI는 사용자가 운동시설 정보를 편하게 읽고 빠르게 판단할 수 있도록 **밝은 화이트/블루 기반 UI**를 적용한다.

### 13-1. 전체 디자인 방향

| 항목 | 내용 |
|---|---|
| 메인 분위기 | 밝은 화이트 배경과 블루 포인트 중심 |
| 강조 카드 | 진한 블루-퍼플 그라데이션 히어로 카드 |
| 카드 스타일 | 흰색 카드 + 부드러운 그림자 |
| 강조 색상 | 블루는 주요 버튼, 민트는 시니어/영업중/긍정 상태 표시 |
| 정보 구조 | 한 화면에 핵심 정보만 배치하여 모바일 가독성 확보 |

### 13-2. 화면별 UI 스타일

| 화면 | UI 스타일 |
|---|---|
| 시작 화면 | 밝은 배경, 큰 서비스 문구, 명확한 시작 버튼 |
| 홈 화면 | 화이트 배경 위에 블루 히어로 카드와 설정 카드 배치 |
| 1차 탐색 화면 | 거리 내 후보를 카드형 목록으로 표시 |
| 추천 조건 화면 | 선호 업종과 운동 목적을 칩 버튼으로 구성 |
| 최종 추천 결과 화면 | TOP 3 추천 카드와 추천 이유 박스 분리 |
| 상세 정보 화면 | 상태, 전화/관리, 위치, 데이터 정보를 각각 카드로 분리 |
| MY 화면 | 자주 쓰는 추천 조건을 프로필 카드 형태로 저장 |

### 13-3. 색상 설계

| 색상 | 사용 위치 | 목적 |
|---|---|---|
| 화이트 | 전체 배경, 정보 카드 | 가독성 확보 |
| 블루 | 주요 버튼, 선택 상태, 위치 아이콘 | 핵심 행동 유도 |
| 블루-퍼플 그라데이션 | 히어로 카드 | 서비스의 기술적 이미지 강조 |
| 민트 | 영업중, 시니어, 공공시설 태그 | 긍정 상태와 안전한 운동 이미지 전달 |
| 다크 네이비 | 로고 아이콘, 일부 CTA | 브랜드 중심성 확보 |

### 13-4. UI 설계 의도

헬스맵 AI는 단순히 시설 목록을 보여주는 앱이 아니라, 사용자가 운동을 시작하기 전 필요한 정보를 한 번에 이해할 수 있도록 설계된 추천형 모바일 서비스이다.

따라서 UI는 다음 세 가지 원칙을 따른다.

1. 첫 화면에서 서비스 목적을 명확히 전달한다.
2. 이후 화면에서는 밝고 읽기 쉬운 카드형 구조를 유지한다.
3. 블루와 민트 포인트를 반복 사용하여 화면 간 통일감을 만든다.

## 14. 설계 포인트

| 설계 포인트 | 내용 |
|---|---|
| 정보 흐름 단순화 | 시작 화면 → 홈 → 1차 탐색 → 조건 선택 → 추천 결과 → 상세 정보 |
| 추천 전 후보 공개 | 최종 추천 전에 거리 안 후보를 먼저 보여줘 신뢰성 확보 |
| 설명 가능한 추천 | 추천 이유를 자연어로 제공하여 사용자가 결과를 이해 가능 |
| 연령대 확장 | 20~40대 헬스장 이용자뿐 아니라 시니어 야외운동 이용자까지 포함 |
| 공공데이터 활용성 | 단순 데이터 조회가 아니라 생활형 추천 서비스로 변환 |
| MVP 구현 가능성 | 위치 계산, 조건 필터링, 카드 UI 중심으로 구현 가능한 구조 |
